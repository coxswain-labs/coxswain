Coxswain has been deployed!

Default topology: two Deployments in {{ include "coxswain.namespace" . }} —
  - {{ include "coxswain.controller.fullname" . }} (leader-elected status writer)
  - {{ include "coxswain.sharedProxy.fullname" . }} (read-only data plane)

Verify both pods are ready:

  kubectl -n {{ include "coxswain.namespace" . }} get pods -l app.kubernetes.io/name={{ include "coxswain.name" . }}
  kubectl get gatewayclass {{ .Values.gatewayClass.name }}

Check controller readiness:

  kubectl -n {{ include "coxswain.namespace" . }} port-forward svc/{{ include "coxswain.controller.fullname" . }} {{ .Values.health.port }}:{{ .Values.health.port }} {{ .Values.admin.port }}:{{ .Values.admin.port }}
  curl http://localhost:{{ .Values.health.port }}/readyz
  curl http://localhost:{{ .Values.admin.port }}/status

Check shared-proxy readiness:

  kubectl -n {{ include "coxswain.namespace" . }} port-forward svc/{{ include "coxswain.sharedProxy.fullname" . }}-internal {{ .Values.health.port }}:{{ .Values.health.port }} {{ .Values.admin.port }}:{{ .Values.admin.port }}
  curl http://localhost:{{ .Values.health.port }}/readyz
  curl http://localhost:{{ .Values.admin.port }}/routes

{{- if .Values.security.rootless }}

Rootless mode is enabled: the shared-proxy container binds 8080/8443 and the
gateway Service maps 80/443 → 8080/8443. CAP_NET_BIND_SERVICE is not required.
{{- end }}

The shared-proxy ServiceAccount holds no Kubernetes API access — it exists
only as a pod identity for mTLS bootstrap. Verify no coxswain RBAC is bound:

  kubectl auth can-i --list --as=system:serviceaccount:{{ include "coxswain.namespace" . }}:{{ include "coxswain.sharedProxy.serviceAccountName" . }}

The output should show only Kubernetes baseline grants (selfsubject* resources
and non-resource URLs). If you see any resource rows beyond those, file a bug.
